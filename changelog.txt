# Zelx Language Changelog

## [Unreleased] - 2026-07-05

### Added
- **Conditional Control Flow (`if`, `else if`, `else`):** Added support for conditional branching blocks using curly braces `{}`. Supports nested and flat `else if` chains.
- **Comparison Operators:** Added support for equality (`==`), inequality (`!=`), less than (`<`), greater than (`>`), less than or equal (`<=`), and greater than or equal (`>=`) operations.
- **Parentheses Grouping:** Added expression grouping support `(value)` to allow overriding operator precedence.
- **Runtime Expression Evaluation:** Implemented `eval_binary_op` in the C runtime to safely compare values dynamically (such as string content checking via `strcmp` and mixed integer/float comparisons).
- **Runtime Truthiness Evaluation:** Implemented `is_truthy` to check the truthiness of any value type dynamically.
- **Shell Command Execution (`exec`):** Added the built-in function `exec(cmd_string)` which runs arbitrary terminal shell commands at runtime (mapping to C's `system()`) and returns the command's exit status.
- **Arithmetic & String Addition (`+`):** Added support for the `+` operator. Resolves to standard math addition for numeric types (`int` and `float`) and dynamic string concatenation (auto-formatting other types like numbers or booleans to string representation) if either operand is a string.
- **Test Case `05_if_else.zx`:** Added to verify conditional logic and comparison operations.
- **Test Case `06_exec_concat.zx`:** Added to verify shell commands execution, exit codes, and string concatenation.

## [Unreleased] - 2026-07-04

### Added
- **Command Line Arguments Support:** Exposed built-in `args` array variable initialized dynamically with `argc`/`argv` (excluding the binary name).
- **Array Indexing:** Added parser grammar and runtime support for retrieving array elements by index (`IDENTIFIER[index]`), working for both user-defined arrays and the built-in `args`.
- **Recursive Array Runtime:** Upgraded the C runtime (`Value` struct) to support actual dynamic arrays containing nested elements.
- **Recursive Array Printer:** Refactored C runtime printing to format and print arrays and their elements recursively (e.g. `["cli", "tools"]`).
- **Array Safety Checks:** Added a runtime safety helper (`get_array_element`) to enforce bounds checking on array reads and prevent segmentation faults.
- **Built-in `print(...)` function:** Implemented modern, Python-like printing. Supports multiple comma-separated arguments, smart unquoting for direct string output (while retaining quotes for strings nested inside arrays), separated by spaces and ending with a newline.
- **Variable Evaluation:** Added support for variable references as values in assignments (`y = x`).
- **Test Infrastructure:** Created `test/valid/` directory containing modular test cases (`01_basic.zx`, `02_array.zx`, `03_multi_assign.zx`, `04_args.zx`), and added a colored shell test runner script `run_tests.sh` along with a conflict-free `Makefile`.
- **Source Line Mapping:** Integrated C preprocessor `#line` directives into the code generation. Tracks original `.zx` line numbers and filenames dynamically (via Flex's `yylineno`), mapping transpiled C code blocks back to their original source lines for seamless debugging in `gdb` and precise compilation warnings/errors.

### Fixed
- **Multi-assignment Bug:** Fixed a Segmentation Fault (exit code 139) caused by a communication mismatch between the parser and codegen.
- **Dynamic Multi-assignment:** Upgraded multi-assignment to support any length of variables (2, 3, etc.) recursively and safely.
- **Type-Aware Multi-assignment:** Multi-assignment now supports mixed dynamic types (`int`, `float`, `string`, `bool`, etc.) using a recursive codegen helper instead of defaulting to integers.
- **Removed Placeholders:** Deleted the dummy code block placeholder for array initialization and replaced it with full, recursive transpilation.
